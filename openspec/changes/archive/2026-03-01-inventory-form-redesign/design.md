## Context

当前库存表单采用简单的两列网格布局，使用 Card 组件包裹表单字段。虽然功能完整，但缺乏视觉层次和设计感。用户希望重新设计表单，增加产品图片展示区域（正方形），并提升整体视觉体验。

现有技术栈：
- SvelteKit + TypeScript
- Tailwind CSS + shadcn-svelte UI 组件库
- SuperForms 表单验证
- 已有 `ImageUploader` 组件支持图片上传

## Goals / Non-Goals

**Goals:**
- 重新设计表单布局，建立清晰的视觉层次
- 添加正方形产品图片展示/上传区域
- 保持与现有 UI 组件库和设计规范的一致性
- 提升表单的用户体验和美观度

**Non-Goals:**
- 不更改表单的业务逻辑和数据验证规则
- 不重构后端 API 或数据库模型
- 不修改其他页面的表单设计（仅限库存表单）

## Decisions

### 1. 表单布局结构

**决定**: 采用左右分栏布局，左侧为正方形图片区域，右侧为表单字段

**理由**: 
- 产品图片是重要的视觉元素，放在醒目位置
- 左右分栏符合用户从左到右的阅读习惯
- 在移动端可堆叠为上下布局，保持响应式

**替代方案**: 图片放在表单顶部
- 否决原因: 占用过多垂直空间，桌面端不够紧凑

### 2. 图片展示比例

**决定**: 使用 1:1 正方形图片区域，使用 `aspect-square` Tailwind 类

**理由**:
- 用户明确要求正方形图片
- 统一的比例便于在列表中展示
- 适配电商产品图片的常见规格

### 3. 表单字段分组

**决定**: 将字段分为三组 - 基本信息、价格信息、库存信息

**理由**:
- 建立逻辑分组，便于用户理解
- 使用小标题或分隔线区分分组
- 减少视觉负担

### 4. ImageUploader 组件增强

**决定**: 扩展现有 ImageUploader 组件，添加 `aspectRatio` prop 支持正方形模式

**理由**:
- 复用现有组件，减少代码重复
- 通过 prop 控制比例，保持组件的灵活性
- 正方形预览与实际展示一致

## Risks / Trade-offs

- **[布局复杂度增加]** → 通过使用 Tailwind 的响应式类和 Grid 布局简化实现
- **[移动端体验]** → 使用 `md:` 断点，移动端堆叠布局，图片在上
- **[ImageUploader 修改影响其他使用处]** → 使用可选 prop，默认行为不变
