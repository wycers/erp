## Context

当前 ERP 系统使用 Drizzle ORM 和 PostgreSQL 数据库。原料和成品分别存储在 `material_sku` 和 `finished_product` 表中，这两个表目前都没有单位字段。

现有技术栈：
- 前端：SvelteKit + shadcn-svelte UI 组件
- 后端：SvelteKit Server Actions
- 数据库：PostgreSQL + Drizzle ORM
- 位置：`src/lib/server/db/erp.schema.ts` 定义表结构

## Goals / Non-Goals

**Goals:**
- 为原料和成品添加可选的单位字段
- 保持向后兼容，现有数据不受影响
- 提供简单的自由文本输入方式

**Non-Goals:**
- 不实现单位标准化或单位换算功能
- 不创建单独的单位主数据表
- 不强制要求填写单位

## Decisions

### 1. 使用可选文本字段而非枚举或外键

**选择**: 使用 `text` 类型的可空字段  
**原因**: 
- 单位种类多样（米、个、件、箱、公斤、升等），难以穷举
- 用户可能有自定义单位需求
- 简单直接，降低实现复杂度

**备选方案**: 
- 创建 `unit` 主数据表 - 增加维护成本，过度设计
- 使用枚举 - 不够灵活，添加新单位需要迁移

### 2. 字段命名为 `unit`

**选择**: 字段名 `unit`，数据库列名 `unit`  
**原因**: 简洁明了，符合业界惯例

### 3. 前端输入方式

**选择**: 简单的 Input 文本框  
**原因**: 
- 与现有表单风格一致
- 实现简单
- 后续可考虑添加常用单位的下拉建议

## Risks / Trade-offs

**[数据不一致]** → 用户可能输入不同写法（如"米"和"m"）  
缓解: 可在未来版本添加输入建议或标准化，当前阶段接受此风险

**[空值处理]** → 历史数据和新数据可能没有单位  
缓解: 字段设为可选，UI 中单位为空时不显示或显示"-"
