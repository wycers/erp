## Context

当前系统中状态标签以英文形式直接显示在界面上：
- ERP 模块：直接显示数据库枚举值（DRAFT、POSTED）
- 订单模块：使用 `statusLabels` 对象映射，但值为英文（Draft、Pending 等）

状态标签分散在多个组件中重复定义，缺乏统一管理。

## Goals / Non-Goals

**Goals:**
- 将所有状态标签翻译为中文显示
- 创建集中管理的状态标签映射
- 保持数据库存储值不变（仍使用英文枚举）

**Non-Goals:**
- 不涉及完整的 i18n 国际化支持
- 不改变数据库 schema 或 API 返回格式

## Decisions

### 决策 1：创建统一的状态标签映射文件

**选择**：在 `src/lib/utils/status-labels.ts` 创建集中管理的映射

**理由**：
- 避免在每个组件中重复定义
- 便于维护和扩展
- 支持多种状态类型（文档状态、订单状态等）

**替代方案**：
- 在各组件中直接定义 → 导致重复代码，维护困难
- 使用 i18n 库 → 过于复杂，当前只需中文

### 决策 2：使用 TypeScript 类型约束

**选择**：为每种状态类型定义独立的映射函数，使用类型参数确保类型安全

**理由**：
- 避免传入错误的状态值
- IDE 自动补全支持

## Risks / Trade-offs

**[硬编码中文]** → 如果将来需要多语言支持，需要重构为 i18n 方案。当前接受此限制，优先满足中文用户需求。

**[映射不完整]** → 如果有新状态值添加但未更新映射，会显示原始英文值。通过 TypeScript 类型检查缓解。
